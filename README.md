# 任务栏歌词 (Taskbar Lyric)

> EchoMusic 插件 - 在 Windows 任务栏上方显示当前播放歌词和专辑封面的透明浮窗

![Version](https://img.shields.io/badge/version-1.4.0-blue)
![EchoMusic](https://img.shields.io/badge/EchoMusic-%3E%3D2.2.6--beta.9-green)

## 功能特性

- 🎵 **卡拉OK双色渐变歌词** — 已播放歌词使用高亮颜色，未播放部分使用暗色，支持逐字（YRC）和整行两种着色模式
- 🎮 **悬停控件栏** — 鼠标移入浮窗时显示播放控制（上一首/暂停/下一首/喜欢/锁定），液态玻璃半透风格，移出后淡出
- ❤️ **喜欢/收藏按钮** — 一键切换喜欢状态，实时从主程序同步快照
- 📀 **专辑封面显示** — 支持圆形/方形封面，可放在歌词左侧或右侧
- 🌐 **翻译与音译** — 当音乐源提供翻译或音译时，可在副行显示
- 🔍 **歌词正则过滤** — 用正则式自动隐藏制作信息、版权信息等无关歌词行
- 📜 **JS 进度驱动滚动** — 歌词过长时按播放进度平滑横向滚动，替代 CSS 动画
- 🔒 **锁定/解锁拖拽** — 锁定后鼠标穿透（mousemove 仍转发用于悬停检测），不干扰正常操作；解锁后可自由拖拽调整位置
- 📍 **任务栏自动贴靠** — 自动检测 Windows 任务栏位置（上/下/左/右），浮窗始终贴靠任务栏边缘
- 🔤 **系统字体扫描** — 通过 EchoMusic 内置 `ctx.fonts` API 扫描系统中已安装的字体
- 🎨 **颜色自定义** — 已播放颜色和未播放颜色均可调
- 📏 **窗口尺寸可调** — 宽度 200–600px，高度 24–80px
- 📐 **手动偏移微调** — 百分比偏移，自动适配不同屏幕分辨率
- 📌 **始终置顶** — 多重保活机制，确保浮窗始终可见

## 安装

1. 将 `taskbar-lyric` 文件夹放入 EchoMusic 的 `plugins` 目录
2. 打开 EchoMusic → 设置 → 插件 → 启用「任务栏歌词」
3. 开始播放音乐，浮窗会自动出现在任务栏上方

## 设置说明

### 启用
| 设置项 | 说明 | 默认值 |

| 启用任务栏歌词 | 控制浮窗的显示与隐藏 | ✅ 开启 |

## 使用技巧

### 悬停控件
1. 将鼠标移到任务栏歌词浮窗上
2. 歌词和封面淡出，显示播放控件栏
3. 可直接操作：⏮ 上一首 / ⏯ 播放/暂停 / ⏭ 下一首 / ❤️ 喜欢 / 🔒 锁定
4. 鼠标移出 300ms 后控件淡出，歌词恢复显示

### 喜欢/收藏
- ❤️ 按钮状态实时同步主程序的喜欢状态
- 点击切换喜欢，主程序同步更新
- 喜欢时显示实心红色（#ff2d55），未喜欢时显示空心

### 歌词过滤
1. 在设置中开启「启用歌词过滤」
2. 在「过滤正则式」输入用 `|` 分隔的关键词
3. 匹配的歌词行将自动隐藏，不影响其他行和时间同步

### 拖拽调整位置
1. 点击悬停控件栏中的 🔒 按钮解锁
2. 按住浮窗任意位置拖拽到目标位置
3. 再次点击 🔒 锁定，系统自动保存偏移量

### 偏移微调
- 偏移值为**百分比**，自动适配不同屏幕分辨率
- 水平偏移：正数向右、负数向左
- 垂直偏移：正数向远离任务栏方向、负数向靠近任务栏方向
- 需勾选「手动调整」后才能使用偏移滑块

### 字体配置
1. 点击「扫描系统字体」按钮
2. 从扫描结果中选择所需字体
3. 点击「保存」生效
4. 首次扫描可能需要几秒钟

### 兼容性要求
- **EchoMusic**：≥ 2.2.6-beta.9
- **操作系统**：Windows 10 (2004+) / Windows 11
- 运行于 EchoMusic 内置的 Electron 42 内核

## 注意事项

⚠️ **任务栏自动检测**
插件通过调用系统接口检测任务栏位置和尺寸。如果使用了第三方任务栏美化工具（如 StartAllBack、ExplorerPatcher），可能导致检测偏差，请使用手动偏移微调。

⚠️ **全屏应用**
全屏游戏或视频播放时，浮窗仍会显示在任务栏上方。如需在全屏时隐藏，请在设置中关闭「启用任务栏歌词」。

⚠️ **多显示器**
插件会在**主显示器**的任务栏上方显示。如需在其他显示器显示，请在 Windows 设置中调整主显示器。

⚠️ **字体扫描**
字体扫描功能通过 EchoMusic 内置的 `ctx.fonts.getAll()` API 获取系统字体列表，无需手动解析 TTF/OTF 文件。

⚠️ **性能**
浮窗使用多重保活机制（每秒置顶断言 + 心跳检测），CPU 占用极低（< 0.1%），不影响正常使用。

## 技术架构

### 文件结构
```
taskbar-lyric/
├── index.js                 # 主入口：设置面板、窗口管理、设置同步
├── taskbar-lyric-window.js  # 浮窗窗口：任务栏检测、歌词渲染、跑马灯
├── taskbar-lyric-window.css # 浮窗样式：淡入过渡、跑马灯动画
├── shared.js                # 共享常量与工具函数：字体扫描、位置计算
├── manifest.json            # 插件清单
├── icon.svg                 # 插件图标
└── README.md                # 本文档
```

### 关键技术点
- **卡拉OK歌词着色**：逐字模式（YRC）按字符起止时间逐字渐变；非逐字模式按整行持续时间整体渐变
- **JS 进度驱动滚动**：33ms 刷新率按播放进度 `translateX` 驱动歌词横向滚动，替代 CSS animation 避免动画与播放不同步
- **悬停控件**：`setIgnoreMouseEvents(true, { forward: true })` 保持 mousemove 接收实现悬停检测，300ms 离开防抖；液态玻璃样式（`backdrop-filter: blur(8px)` + 半透深色背景）
- **喜欢状态**：直接从 `snapshot.playback.isFavorite` 读取（EchoMusic >= 2.2.6 主进程维护），发送 `toggleFavorite` 命令切换
- **歌词过滤**：`lyricFilterEnabled` + `lyricFilterPatterns` 正则匹配隐藏无关行，`findNextVisibleIndex` 降级策略保证时间同步
- **设置同步**：优先使用 BroadcastChannel 跨进程同步，不可用时回退到 storage 轮询（1s 间隔，hash 比较避免重复赋值）
- **常驻保活**：三重防御 — 浮窗每 3s 断言 `setAlwaysOnTop`、每 2s 写心跳时间戳到 storage 供主进程检测、主进程每 1s 调用 `recoverWindow`
- **字体扫描**：通过 `ctx.fonts.getAll()` API 获取系统字体列表，无需手动解析 TTF/OTF 二进制文件
- **位置锚定**：宽度变化时以封面侧边缘为锚点，防止 Electron 默认左边缘对齐导致的中心漂移
- **偏移百分比**：偏移值以窗口可移动范围为基准的百分比，自动适配不同屏幕分辨率
